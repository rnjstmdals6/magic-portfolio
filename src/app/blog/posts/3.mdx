# 토티 - 개발자 스터디 및 멘토링 플랫폼

## 프로젝트 목적

IT 관련 종사자 및 학습자들은 스터디와 멘토링을 통해 성장할 기회를 필요로 합니다. 하지만, 기존의 스터디 모집 및 멘토링 연결 시스템은 분산되어 있어 불편함이 많습니다. 토티는 이러한 문제를 해결하기 위해 개발자와 IT 학습자를 대상으로 스터디를 도모하고 멘토링을 연결하는 웹 기반 서비스입니다.

코로나로 인해 대학 시절 멘토를 만들기 어려웠던 경험이 아쉬웠고, 스터디 및 멘토링을 더 쉽게 연결할 수 있는 플랫폼이 필요하다고 느꼈습니다. 이를 바탕으로 개발자와 IT 학습자들이 보다 효과적으로 학습하고 성장할 수 있도록 돕기 위해 이 프로젝트를 기획하게 되었습니다.
## 핵심 목표

- **카카오 및 구글 소셜 로그인 지원**
- **스터디 및 멘토링 모집 기능**
- **댓글 및 알림 기능 제공**
- **마이페이지를 통한 스터디 및 멘토링 멤버 관리**

## 프로젝트 개요

- **프로젝트명**: 토티
- **설명**: 개발자 및 IT 학습자를 위한 스터디 및 멘토링 플랫폼
- **도메인**: 운영 중단
- **깃허브**: [프로젝트 GitHub 링크](https://github.com/Totee-Side-Project/Totee-WEB-Backend)
- **작업 인원**: 백엔드 1명(본인) / 프론트엔드 2명 / 디자이너 2명

## 주요 기능

### 1. 카카오 및 구글 소셜 로그인 기능
- OAuth를 활용한 간편 로그인 구현
- 사용자 정보 보호 및 편리한 인증 제공

### 2. 스터디 및 멘토링 모집 기능
- 스터디 및 멘토링 개설 및 지원 기능
- 모집 현황 및 참여자 관리 기능 제공

### 3. 댓글 및 알림 기능
- 사용자 간 소통을 위한 댓글 및 대댓글 기능 제공
- 알림 기능을 통해 모집 및 댓글 활동 확인 가능

### 4. 마이페이지 스터디 및 멘토링 관리 기능
- 사용자가 참여 중인 스터디 및 멘토링 관리
- 모집글 및 지원 내역 확인 가능

## 담당 업무

- **프로젝트 기획 및 팀원 모집**
- **프로젝트 일정 관리 및 백엔드 개발 총괄**
- **CI/CD 구축을 위해 Github Actions 및 Docker 사용하여 AWS EC2 자동 배포 구현**
- **RESTful API 설계 및 구현**:
- 소셜 로그인 / 정보 수정 / 멘토 & 스터디 개설 및 지원
- 댓글 및 대댓글 작성 / 알림 기능 개발

## 성과 및 배운 점

### 1. AWS EC2 환경에서 Docker를 활용한 배포
- CI/CD 환경 구축을 위해 **Docker + Github Actions + AWS EC2** 조합 사용
- 서버 환경을 컨테이너화하여 손쉬운 배포 및 유지보수 가능

### 2. 예외 처리 핸들러 적용
- 예외 발생 시 일관된 HTTP 응답을 제공하도록 **Global Exception Handler** 적용
- 클라이언트가 예외 상태를 쉽게 이해하고 처리할 수 있도록 명확한 상태 코드 반환

### 3. QueryDSL을 활용한 쿼리 최적화
- QueryDSL을 사용하여 **컴파일 시점에서 쿼리 검증**
- 동적 쿼리를 적용하여 유연한 데이터 조회 및 성능 향상

### 4. 팀원 모집부터 운영까지 총괄하며 얻은 리더십 경험
- 프로젝트를 처음부터 기획하고 팀원을 모집하여 **주도적으로 진행**
- 개발뿐만 아니라 팀 내 **일정 조율, 역할 분배 등을 총괄하며 협업 경험을 쌓음**

### 5. 마무리하지 못한 아쉬움과 배운 점
- **첫 기획 프로젝트였던 만큼 어설프고 미숙했으며, 기획 방식도 체계적이지 못해 시행착오가 많았음**
- 하지만 이러한 경험을 통해 프로젝트 관리의 중요성을 깊이 깨닫고, **더 나은 기획 방법과 협업 방식을 학습하는 계기가 되었음**
- **기획 단계에서 사용자 요구사항을 체계적으로 정리하는 중요성을 깨달음**
- 기능 정의서를 작성하고, 우선순위를 정리하여 기획의 방향성을 명확히 할 필요가 있었음
