# GDSC 블로그 프로젝트 포트폴리오

### 프로젝트 목적

기존에는 GDSC 멤버들이 학습한 내용을 체계적으로 공유할 플랫폼이 없어, 개인 블로그 또는 노션을 이용해야 했습니다. 이를 해결하기 위해 GDSC 멤버들이 **카테고리별로 학습 결과물을 게시하고, 쉽게 관리할 수 있는 블로그 플랫폼**을 구축했습니다.

### 핵심 목표

- 포지션별 학습 결과물 게시판 제공
- 카테고리별 포스트 조회 기능
- 관리자 전용 회원 관리 기능
- 게시글 스크랩 기능

## 프로젝트 개요

- 프로젝트명: GDSC 블로그
- 설명: 대진대학교 GDSC(Google Developer Student Club) 멤버 전용 블로그 플랫폼
- 배포 링크: 운영 중단
- 깃허브: [프로젝트 GitHub 링크](https://github.com/GDSC-Daejin/GDSC-WEB-BackEnd)
- **작업 인원**: 백엔드 3명(본인) / 프론트엔드 3명 / 디자이너 3명

## 주요 기능

- 포지션별 학습 결과물 게시판
    - 개발자, 디자이너 등 역할에 따른 학습 기록 공유
    - Markdown 기반 에디터 지원

- 카테고리별 포스트 조회 기능
    - 카테고리별로 포스트를 필터링하여 검색 가능
    - 최신순, 인기순 정렬 기능 지원

- 관리자 전용 회원 관리 기능
    - GDSC 멤버 등록 및 권한 관리 기능
    - 회원 정보 수정 및 삭제 기능 제공

- 게시글 스크랩 기능
    - 관심 있는 게시글을 저장하여 나중에 다시 확인 가능
    - 사용자별 스크랩한 게시글 목록 제공

## 담당 업무

- 관리자 기능 API 개발: GDSC 멤버 관리(등록, 수정, 삭제)
- 카테고리 및 게시글 조회 API 개발: 특정 카테고리별 게시글 필터링 기능 구현
- 게시글 스크랩 기능 개발: 사용자가 원하는 게시글을 저장하고 관리할 수 있도록 API 제공
- 기능별 통합 테스트 코드 작성: JUnit과 MockMVC를 활용한 API 테스트 코드 작성

## 성과 및 배운 점

### 첫 번째 백엔드 프로젝트 & 협업 프로젝트 경험
이 프로젝트는 **제가 처음으로 참여한 백엔드 개발 및 협업 프로젝트**였습니다.
기존에는 개인 프로젝트 위주로 개발을 진행했지만, 이번 프로젝트에서는 다른 개발자 및 디자이너와 협업하며 실무에 가까운 경험을 쌓을 수 있었습니다.
- Git & GitHub을 활용한 협업
- PR 리뷰 및 코드 리뷰를 진행하며 팀원들과 원활하게 협력하는 방법을 익혔습니다.
- **백엔드 개발 프로세스에 대한 실질적인 이해**
- 요구사항 분석 → API 설계 → 개발 → 테스트 → 배포까지의 흐름을 경험하며, 백엔드 개발의 전체적인 사이클을 익힐 수 있었습니다.
- 단순히 기능 구현을 넘어서, 확장성과 유지보수성을 고려한 코드 작성을 고민하는 계기가 되었습니다.

---

### Spring Boot 기반 CRUD API 설계 경험
이 프로젝트에서는 Spring Boot를 활용하여 회원, 게시글, 카테고리 등의 CRUD API를 개발하였으며, RESTful한 API 설계를 적용하였습니다.
- Controller - Service - Repository 계층을 분리하여 설계
- 역할에 맞게 책임을 분리하여 유지보수성을 높이고, 코드의 가독성을 향상시켰습니다.
- RESTful API 원칙을 준수하면서도 비즈니스 로직을 명확하게 분리하는 방법을 익혔습니다.

- DTO를 활용한 요청 및 응답 데이터 처리
- 엔티티와 DTO를 명확하게 분리하여, 불필요한 데이터 노출을 방지하고 유지보수성을 높이는 방법을 배웠습니다.

---

### JPA를 활용한 객체 간 관계 설정 및 최적화
- 게시글과 카테고리 간의 관계를 JPA의 `@ManyToMany` 매핑을 통해 구현했습니다.
- 하지만 성능 최적화를 위해 중간 테이블을 따로 분리하여 N:1, 1:M 구조로 변환, 더 효율적인 쿼리를 구성할 수 있도록 개선하였습니다.
- JPA의 즉시 로딩(Eager)과 지연 로딩(Lazy)의 차이점을 학습하고, 성능 최적화를 위해 Lazy 로딩을 적극 활용하는 방법을 익혔습니다.

---

### 쿠키를 활용한 조회 수 중복 증가 문제 해결
- 초기에 게시글 조회 기능을 구현했을 때, 사용자가 새로고침할 때마다 조회 수가 증가하는 문제가 발생했습니다.
- 이를 해결하기 위해 쿠키를 활용하여 사용자별 조회 여부를 저장하고, 동일 사용자가 일정 시간 내에 같은 게시글을 재조회할 경우 조회 수가 증가하지 않도록 처리하였습니다.
- HTTP의 상태 관리 방식 Cookie에 대한 이해도를 높이는 계기가 되었습니다.

### 기능별 테스트 코드 작성 및 코드 안전성 향상
- 프로젝트 후반부에서는 JUnit과 MockMVC를 활용하여 API 단위 테스트 및 통합 테스트를 작성했습니다.
- 특히, 예외 처리 및 경계값 테스트를 강화하여 예상치 못한 오류를 방지하고, 서비스의 안정성을 높이는 데 집중했습니다.

### 프로젝트를 통해 성장한 점
이 프로젝트를 통해 단순히 백엔드 개발 기술뿐만 아니라, 협업과 실무 개발 경험, 문제 해결 능력을 기를 수 있었습니다.
- 백엔드 개발의 전반적인 흐름을 경험하면서, 실무에서의 개발 프로세스를 이해하는 계기가 되었습니다.
- 협업을 통해 의사소통의 중요성을 배우고, 코드 리뷰를 통해 더 나은 코드 스타일과 개발 방식을 익혔습니다.
- 여러 가지 문제를 해결하면서 기능 개발에서 발생할 수 있는 다양한 이슈에 대한 대응 능력을 키웠습니다.

이 프로젝트는 단순한 기능 구현을 넘어서, 협업 속에서 성장할 수 있었던 경험이자, 백엔드 개발자로서 한 단계 도약할 수 있었던 중요한 계기가 되었습니다.
