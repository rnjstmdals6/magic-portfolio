# 메모톡 프로젝트 포트폴리오

### 프로젝트 목적

기존 메모 애플리케이션은 단순한 텍스트 저장 기능만을 제공하여 실시간 기록 및 관리가 어렵고, 일정 관리 기능이 부족했습니다. 또한, 메모와 일정이 별도로 관리되어 사용성이 떨어지는 문제가 있었습니다. 이를 해결하기 위해 **채팅 형식으로 메모를 기록하고 일정 관리까지 함께 할 수 있는 웹 애플리케이션**을 개발하였습니다.

### 핵심 목표

- 실시간 메모 작성 및 저장 기능 제공
- Todo List 기반 일정 관리 기능 추가
- 이메일 인증을 통한 보안 강화
- 이미지 업로드를 통한 시각적 메모 지원

## 프로젝트 개요

- **프로젝트명:** 메모톡
- **설명:** 나만의 채팅방에서 메모를 하고 일정을 관리할 수 있는 심플하고 편리한 웹 애플리케이션
- **배포 링크:** 운영 중단
- **깃허브:** [프로젝트 GitHub 링크](https://github.com/memo-talk/memotalk-webserver)
- **작업 인원**: 백엔드 1명(본인) / 프론트엔드 2명 / 디자이너 1명

## 주요 기능

###### 회원가입 및 인증 시스템
- 이메일 인증을 기반으로 회원가입 기능을 제공하여 보안성을 강화하였습니다.
- Redis를 활용하여 인증 코드를 저장하고 TTL(Time To Live) 설정을 적용하여 일정 시간이 지나면 자동 만료되도록 구현하였습니다.

###### 채팅 형식의 메모 작성 기능
- 사용자는 자신만의 채팅방을 생성하고, 실시간으로 메모를 기록할 수 있습니다.
- WebSocket 및 STOMP 프로토콜을 활용하여 실시간 양방향 데이터 전송을 지원하였습니다.

###### Todo List 기반 일정 관리 기능
- 사용자는 자신이 생성한 채팅방 내에서 일정 관리가 가능한 Todo List를 작성할 수 있습니다.
- 할 일 추가, 완료, 삭제 기능을 제공하여 사용자가 편리하게 일정 관리를 할 수 있도록 구현하였습니다.

###### 이미지 업로드 기능
- 사용자는 채팅방 내에서 이미지를 업로드하여 텍스트와 함께 저장할 수 있습니다.

## 담당 업무

- 메모 및 Todo List 기능 개발
- 채팅방 이미지 업로드 기능 개발
- 백엔드 개발 100% 담당

## 성과 및 배운 점

### Redis를 활용한 이메일 인증 성능 최적화
- 이메일 인증 요청 시 기존 데이터베이스 접근을 최소화하기 위해 **Redis를 캐시 서버로 활용**하였습니다.
- TTL(Time To Live) 설정을 적용하여 일정 시간이 지나면 인증 코드가 자동 만료되도록 구현하였습니다.

### STOMP 프로토콜을 활용한 실시간 양방향 통신
- WebSocket을 활용하여 **채팅 형식의 메모 작성 기능**을 구현하였으며, STOMP 프로토콜을 사용하여 **클라이언트와 서버 간 실시간 양방향 통신을 안정적으로 처리**할 수 있도록 설계하였습니다.
- 기존의 HTTP 기반 통신 방식과 달리, **이벤트 기반의 실시간 데이터 흐름을 경험할 수 있었으며, WebSocket의 연결 관리 및 메시지 전송 방식에 대한 이해도를 높일 수 있었습니다.**
- STOMP 프로토콜을 적용함으로써 **메시지 브로커(Pub-Sub 구조)를 활용한 효율적인 데이터 송수신**을 경험하였습니다.

### JIB를 활용한 Docker 이미지 빌드 자동화
- JIB를 사용하여 **Docker 이미지 빌드 및 배포 과정을 간소화**하였으며, 컨테이너 환경에서 백엔드 애플리케이션을 효율적으로 운영할 수 있도록 설정하였습니다.
- 기존 Dockerfile을 활용한 빌드 방식보다 **더 빠르고 가벼운 이미지 빌드를 지원**하여 배포 속도를 개선할 수 있었습니다.
- JIB의 계층화된 빌드 구조를 통해, **애플리케이션 코드 수정 시 불필요한 레이어 빌드를 방지하여 빌드 성능을 최적화**할 수 있었습니다.
